# Bilillee Build Phases

This file is the working delivery plan for the app. We will refine features against these phases instead of making disconnected changes.

## Phase 1: MVP Foundation

Goal: make the product usable end to end for the core marketplace flow.

### Public / Customer
- Refine homepage discovery and search
- Refine shop list cards and trust signals
- Refine public shop page structure
- Make booking flow clear and usable
- Show payment options clearly
- Show booking history clearly
- Improve mobile responsiveness

### Shop Owner
- Refine owner dashboard overview
- Refine service management
- Refine booking calendar and daily operations
- Refine client directory
- Refine shop profile basics
- Show subscription usage clearly

### Staff
- Refine staff dashboard basics
- Show assigned bookings clearly
- Show visible public profile expectations
- Show portfolio and review basics

### Super Admin
- Refine platform overview
- Refine shop approval workflow
- Refine subscription plan visibility
- Refine platform metrics summary

### Cross-Cutting
- Unify role language across the app
- Remove obvious placeholder UX where possible
- Improve responsive layout system
- Stabilize key UI states: loading, empty, warning, blocked

## Phase 2: Operational Depth

Goal: make the system strong for day-to-day business use.

- Add richer service CRUD flows
- Add better staff assignment and permission flows
- Add stronger booking status management
- Add owner review management
- Add better portfolio workflows
- Add smarter filtering, sorting, and search
- Improve analytics depth
- Improve notifications UX

## Phase 3: Production Readiness

Goal: make the system resilient, auditable, and launch-ready.

- Validate schema alignment fully
- Tighten RLS and role protections
- Test all routes and role access
- Test booking limits and plan blocking
- Add better error handling and validation
- Reduce bundle size and improve performance
- Finalize copy, polish, and consistency
